# shared/

跨任务共享的工具代码库。

## 放什么
- 配置加载工具
- 统一的日志封装
- 常用的数据结构、通用 utils
- 多处复用的小函数/类

## 不放什么
- 任何只被一个子目录使用的代码（放到对应子目录）
- 复杂的业务逻辑

## 原则

**先三次，再抽象**：代码在 3 个不同地方用到了，再考虑抽到 `shared/` 里。不要过早抽象。

## 组织方式

按功能分模块：

```
shared/
├── config/          # 配置加载
├── logging/         # 日志封装
└── utils/           # 通用小工具
```
